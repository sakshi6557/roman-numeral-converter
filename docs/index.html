Roman Numeral Converter

A web service and UI application that converts numbers to Roman numerals, built with TypeScript, React, and Adobe's React Spectrum library.

Project Overview

This project consists of two main components:

  1. A backend web service that handles the Roman numeral conversion
  2. A frontend React application with a modern UI built using Adobe's React Spectrum components

Project Structure

roman-numeral-converter/
├── backend/                 # Backend service
│   ├── src/
│   │   ├── index.ts        # Main server file
│   │   ├── converter.ts    # Roman numeral conversion logic
│   │   └── logger.ts       # Logging configuration
│   ├── package.json
│   └── Dockerfile
├── frontend/                # React application
│   ├── src/
│   │   ├── components/     # React components
│   │   ├── App.tsx         # Main application component
│   │   └── main.tsx        # Application entry point
│   ├── package.json
│   └── Dockerfile
└── docker-compose.yml       # Container orchestration

Technology Stack and Dependencies

Backend

  • Express.js: Chosen for its simplicity, performance, and extensive middleware ecosystem
  • TypeScript: Provides type safety and better developer experience
  • Winston: For structured logging and observability
  • Jest: For unit testing
  • Docker: For containerization and deployment

Frontend

  • React: Chosen for its component-based architecture and large ecosystem
  • Adobe React Spectrum: Selected for its accessibility features, theming support, and professional design system
  • Vite: For fast development and optimized production builds
  • Vitest: For testing React components
  • Axios: For making HTTP requests to the backend service

Dependency Selection Reasoning

  1. Express.js:

    • Lightweight and fast
    • Excellent TypeScript support
    • Large ecosystem of middleware
    • Easy to containerize

  2. Adobe React Spectrum:

    • Built-in accessibility features
    • Comprehensive component library
    • Built-in theming support
    • Professional design system
    • Active maintenance and support

  3. Winston:

    • Structured logging capabilities
    • Multiple transport options
    • Easy integration with monitoring tools
    • Production-ready features

  4. Vite:

    • Fast development server
    • Optimized production builds
    • Excellent TypeScript support
    • Modern build tooling

Building and Running the Project

Prerequisites

  • Node.js (v18 or higher)
  • Docker and Docker Compose
  • Git

Development Setup

  1. Clone the repository:

    git clone https://github.com/yourusername/roman-numeral-converter.git
    cd roman-numeral-converter

  2. Install dependencies:

    # Install backend dependencies
    cd backend
    npm install

    # Install frontend dependencies
    cd ../frontend
    npm install

  3. Start the development servers:

    # Start backend server
    cd backend
    npm run dev

    # Start frontend server
    cd ../frontend
    npm run dev

Production Deployment

  1. Build and run using Docker Compose:

    docker-compose up --build

  2. Access the application at:

    • Frontend: http://localhost:5173
    • Backend API: http://localhost:8080
